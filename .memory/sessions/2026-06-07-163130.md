## Key Decisions

- **Runtime: all TypeScript.** Agent = TypeScript + `@google/adk` (official first-party TS ADK,
  pinned `1.2.0`) + Gemini 2.5 Pro via `@google-cloud/vertexai`. Frontend = React + Vite + TS.
  - Rationale: official first-party TS ADK exists; reuses existing TS tooling; lets the event
    contract be a single shared module imported by agent + web (no pydantic mirror, no drift).
  - Trade-off accepted: ADK-TS is younger than Python ADK, and **Vertex AI Agent Engine is
    Python-only** → deploy agent + frontend both on **Cloud Run** (containers).
- **Transport:** SSE, single stream, JSON discriminated union on `type`. Server framework =
  **Hono** (`4.12.23`) — native SSE helper, Cloud Run-friendly.
- **Evidence richness (Q2):** DQL + scalar summary + deep-link. No inline timeseries on the wire
  (additive `Evidence.series?` later if needed for demo charts).
- **Opportunities (Q3):** stream incrementally via `opportunity.added`, then final `board.ready`.
- Existing `src/` TS stubs are kept and built out as the real agent; Python `.venv` removed.
