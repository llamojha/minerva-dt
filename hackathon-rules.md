## Challenge Goals

1. **Move Beyond Chat** — Agent must use tools and capabilities to accomplish tasks, not just answer questions.
2. **Multi-Step Mission** — Agent must plan steps and use tools to finish a job while keeping the user in control.
3. **Partner Power** — Meaningful integration with Dynatrace MCP server.

## What to Build

A functional agent — powered by **Gemini** and **Google Cloud Agent Builder** — that integrates **Dynatrace's MCP server** to solve a real challenge.

## Technical Requirements

| Requirement | Detail |

## Project Rules

- **New projects only** — must be created during the contest period (May 5 – June 11, 2026).
- **Original work** — not a modification or extension of existing work.
- **Max team size**: 4 individuals.
- **Multiple submissions** allowed if substantially different.
- **Third-party integrations** must be properly licensed/authorized.

## Judging Criteria (Equal Weight)

| Criterion | Question |
|-----------|----------|
| **Technological Implementation** | Does the interaction with Google Cloud and Partner services demonstrate quality software development? |
| **Design** | Is the user experience and design of the project well thought out? |
| **Potential Impact** | How big of an impact could the project have on target communities? |
| **Quality of the Idea** | How creative and unique is the project? |

## Judging Process

1. **Stage One (Pass/Fail)**: Submission includes all requirements, reasonably addresses the challenge, and applies both Dynatrace data and Google Cloud products.
2. **Stage Two (Scoring)**: Evaluated on the 4 criteria above with equal weighting. Ties broken by criterion order, then judge vote.
